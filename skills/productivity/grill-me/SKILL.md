Pull the knowledge out of the user that the docs and code do not contain. Ground every question in what already exists, and ask one question at a time.

Read [the foundation doctrine](../../meta/foundation/SKILL.md) first.

## Steps

1. **Read what exists.** Read the relevant docs and code first. The questions cover the gaps in that material, not what it already answers.

2. **Ask one question.** Pose a single pointed question, then wait for the answer. A batch of questions at once lowers the quality of each answer.

3. **Press on the vague.** When an answer stays abstract, ask for a concrete example or a number. The follow-up turns an opinion into a specification.

4. **Record the answer.** Write each answer into the target document as it arrives. The record is the output, not the conversation.

5. **Stop at saturation.** End once new questions stop changing the document. The grill is done when the spec answers its own open questions.
