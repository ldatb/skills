Turn a live trend into on-brand post ideas for a founder who is growing an audience to sell a product. The leverage is in stealing the *structure* of what is spiking — the hook, the format, the emotional driver — and rebuilding it in the founder's niche and voice, never copying the content.

This skill is browser-driven research plus ideation. The Chrome integration drives the user's already-logged-in Instagram and X to observe what is spiking by hand. POSTING and every engagement action (like, follow, reply, repost, schedule) are external mutations that change the world: stop and get explicit user approval before any one of them. This skill researches and drafts only.

The durable engine — pillars, cadence, the give-before-you-sell ratio — lives in the [social-content](../../marketing/social-content/SKILL.md) skill. Trends are fuel poured into that engine, never a replacement for it. A founder who chases trends without pillars builds reach that converts no one.

Read [the viral-patterns reference](references/viral-patterns.md) before researching: the manual browser research method, the viral-pattern taxonomy (hooks, formats, emotional drivers, save/share triggers), how to adapt a pattern authentically, X-versus-Instagram norms, the post-and-test loop, the ethics and ToS line, the failure modes, red flags, and a worked example.

## Steps

1. **Frame the niche and the goal.** State the founder's niche, the target buyer, the product the audience is being grown to sell, and the one metric this round chases (reach, saves, follows, or clicks). A trend adapted without a niche and a goal is a costume, so name both before opening the browser. This step is done when the niche, the buyer, the product, and one named goal metric are written down.

2. **Scan live trends by hand in the browser.** Drive the user's logged-in Chrome to scan X (trending, search in the niche, the For You feed) and Instagram (Explore, Reels, niche hashtags), browsing manually at human pace per the ethics line in the reference. Capture 8 to 12 spiking posts. This step is done when each captured post records its platform, format, hook line, structure, length, posting time, and engagement signal.

3. **Extract the transferable structure.** Reduce each captured post to its reusable pattern using the taxonomy in the reference: the hook type, the format, the emotional driver, and the save-or-share trigger. Name the underlying pattern, not the surface topic. This step is done when at least 3 captured posts each carry a one-line pattern abstraction stripped of their original subject.

3a. **Score each pattern for fit.** Rate every extracted pattern against the niche, the goal metric from step 1, and the founder's authentic authority. Drop a pattern the founder cannot speak to with earned credibility, and drop one that buys reach the product can never convert. This step is done when each surviving pattern carries a fit verdict tied to the goal metric.

4. **Adapt the pattern to the niche.** Rebuild one surviving pattern as the founder's own — the founder's expertise, the buyer's pain, the founder's voice — keeping the structure and replacing every piece of the borrowed content. Honor the platform norms in the reference: X and Instagram reward different shapes. This step is done when one pattern has spawned 3 distinct niche post ideas, each naming a platform, a format, and a hook line in the founder's voice.

5. **Draft testable variations.** Write each of the 3 ideas as a postable draft, then add one hook variant per draft so the founder can test the hook against itself. Label the draft a draft and leave it unposted, because posting is an external mutation that requires the user's explicit approval. This step is done when 3 drafts exist, each with a primary hook and one alternate hook, all marked unposted.

6. **Lay out the post-and-test plan.** Propose a posting slot and cadence for the drafts on top of the founder's baseline calendar, name the engagement signal to watch per platform, and state the double-down rule for a winner. Present the plan for approval rather than enacting it. This step is done when each draft carries a proposed slot, a named success signal, and a double-down threshold, with posting left pending explicit approval.

7. **Check against the failure modes.** Test the drafted set against the failure-modes and red-flags lists in the reference: off-brand trend-chasing, copying instead of adapting, vanity virality that will not convert, and any ToS-risky automation. This step is done when the set trips zero red flags and every flagged draft is reworked or cut.
